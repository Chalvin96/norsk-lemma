# Google Lemma Audio Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate Google Cloud Text-to-Speech audio for lemma entries, link the audio files from exported lemma JSON, and publish both word data and audio in release archives.

**Architecture:** Add a standalone audio generation command that reads existing `lemma/*.json`, extracts lemma-headword audio jobs, synthesizes MP3 files with Google Cloud Text-to-Speech, writes a manifest, and optionally creates audio-enriched release JSON in a separate output directory. Keep `build.py` as the clean deterministic lemma builder; audio is a second artifact-enrichment stage so credentials, paid API calls, and release packaging are opt-in and resumable.

**Tech Stack:** Python 3.12, `uv`, Google Cloud Text-to-Speech client library, existing `scripts/ordbokene` CLI/test patterns, GitHub Actions release tarball.

---

- Verify Google Cloud Text-to-Speech generated-audio redistribution is acceptable for this project before publishing audio publicly. Google docs describe generated audio as usable to power applications or augment media, but the release must still comply with Google Cloud Platform Terms and applicable law.
- Decide whether generated audio is committed, stored as GitHub release artifacts only, or generated by a private/manual release step. Initial recommendation: do not generate audio in CI; generate locally with credentials, then package as a separate release asset.
- Pick the first Google voice only after a 200-word bakeoff. `nb-NO-Chirp3-HD-Aoede`
  is only an example from Google's current supported-voices list; the release voice
  should be explicit and verified with the live `voices:list` API before generation.

## Data Model

Add this field to each audio-enriched release lemma entry. Keep plain `lemma/{article_id}.json`
available as the clean dictionary artifact; write enriched JSON to
`dist/lemma-with-audio/{article_id}.json` unless the caller explicitly opts into
in-place mutation.

```jsonc
{
  "lemma": "bønner",
  "audio": {
    "lemma": [
      {
        "type": "tts",
        "provider": "google",
        "voice": "nb-NO-Chirp3-HD-Aoede",
        "language_code": "nb-NO",
        "format": "mp3",
        "file": "9e1c4b4f2a7d.mp3",
        "text": "bønner",
        "source": "generated",
        "content_sha256": "9e1c4b4f2a7d...",
        "pronunciation_source": "nb_uttale",
        "tone_status": "known",
        "tone": 2
      }
    ]
  }
}
```

Filename rule:

```text
{12_char_hash}.mp3
```

Hash input:

```text
google|{voice}|{language_code}|{normalized_text}|{tone_status}|{tone}
```

Normalize text by stripping leading/trailing whitespace and collapsing internal whitespace to one space. Do not lowercase; Norwegian capitalization can be meaningful for names.

Use a deterministic hash/UUID-style filename, not a fresh random filename. Random
filenames make regeneration, dedupe, release diffs, and cache invalidation harder.
The MP3 can still be stored under provider/voice directories on disk:

```text
audio/lemma/google/{voice}/{filename}
```

but lemma JSON should only store the filename. Clients combine the filename with
the release/provider base path they choose.

Collision rule:

- If the same normalized lemma text appears with the same tone metadata, reuse one
  audio asset.
- If the same normalized lemma text appears with conflicting known tone metadata,
  create separate jobs keyed by `(text, tone_status, tone)` and include all
  conflicts in the manifest.
- If Google cannot reliably realize the tone difference, keep the separate
  metadata anyway but do not claim the audio is tone-controlled.

Manifest path:

```text
audio/manifest-google-{voice}.json
```

Manifest shape:

```jsonc
{
  "provider": "google",
  "voice": "nb-NO-Chirp3-HD-Aoede",
  "language_code": "nb-NO",
  "format": "mp3",
  "items": [
    {
      "text": "bønner",
      "file": "9e1c4b4f2a7d.mp3",
      "path": "audio/lemma/google/nb-NO-Chirp3-HD-Aoede/9e1c4b4f2a7d.mp3",
      "article_ids": [12345],
      "source_lemma_ids": [67890],
      "tone_status": "known",
      "tone": 2,
      "content_sha256": "9e1c4b4f2a7d...",
      "conflicts": []
    }
  ]
}
```

Dry-run output must include:

- total lemma entries scanned
- unique audio jobs
- jobs with existing MP3 files
- pending API calls
- total characters that would be sent to Google
- estimated cost from configured per-million-character price

Paid full-corpus runs require `--confirm-cost` unless `--limit` is set.

Voice selection must be dynamic:

- Add a `--list-voices` mode that calls Google `voices:list` for `nb-NO`.
- Do not assume WaveNet exists for Norwegian in the user's dashboard.
- Store the selected voice name in the manifest and audio metadata.
- Default docs may show `nb-NO-Chirp3-HD-Aoede` as an example, but the command
  should make `--voice` explicit for release generation.

## Reviewer-Mandated Design Decisions

These changes came from Claude CLI and subagent review and are part of the plan:

- Do not mutate core `lemma/*.json` as the default. Write audio-enriched JSON to
  `dist/lemma-with-audio/` so the clean lemma build stays reproducible.
- Do not dedupe by lemma text alone. Key audio jobs by
  `(normalized_text, tone_status, tone)` so tonal homographs do not silently share
  a single asset.
- Do not generate audio in normal CI. Google credentials and quota must stay out
  of ordinary tests and release builds.
- Do not leave partial MP3 files. Google synthesis must write to a temp file and
  atomically rename after success.
- Do not skip files by existence alone once checksums exist. The manifest should
  carry `content_sha256`, and resumable runs should treat missing/zero-byte or
  checksum-mismatched files as pending.
- Do not bundle audio into the main lemma archive by default. Publish a separate
  audio artifact such as `norsk-lemma-audio-google-${GITHUB_REF_NAME}.tar.gz`.
- Do not run a full paid corpus generation without a dry-run cost report and an
  explicit `--confirm-cost` flag.

```python
from __future__ import annotations

import hashlib
import json
import re
from dataclasses import dataclass, field
from pathlib import Path
from typing import Any

_SPACE_RE = re.compile(r"\s+")


@dataclass
class AudioJob:
    key: tuple[str, str | None, int | None]
    text: str
    provider: str
    voice: str
    language_code: str
    filename: str
    article_ids: list[int] = field(default_factory=list)
    source_lemma_ids: list[int] = field(default_factory=list)
    pronunciation_source: str | None = None
    tone_status: str | None = None
    tone: int | None = None


def normalize_audio_text(text: str) -> str:
    return _SPACE_RE.sub(" ", text.strip())


def audio_filename(
    provider: str,
    voice: str,
    language_code: str,
    text: str,
    tone_status: str | None,
    tone: int | None,
) -> str:
    normalized = normalize_audio_text(text)
    digest = hashlib.sha256(
        f"{provider}|{voice}|{language_code}|{normalized}|{tone_status}|{tone}".encode("utf-8")
    ).hexdigest()
    return f"{digest[:12]}.mp3"


def collect_audio_jobs(
    lemma_dir: Path,
    *,
    provider: str,
    voice: str,
    language_code: str,
    include_expressions: bool = False,
) -> list[AudioJob]:
    jobs_by_key: dict[tuple[str, str | None, int | None], AudioJob] = {}

    for path in sorted(lemma_dir.glob("*.json")):
        data = json.loads(path.read_text(encoding="utf-8"))
        article_id = int(data.get("source_article_id") or path.stem)
        for lemma in data.get("lemmas", []):
            if not isinstance(lemma, dict):
                continue
            text = normalize_audio_text(str(lemma.get("lemma") or ""))
            if not text:
                continue
            if not include_expressions and _is_expression(lemma, text):
                continue

            pron = _best_pronunciation_for_lemma(lemma, text)
            tone_status = pron.get("tone_status") if pron else None
            tone = pron.get("tone") if pron else None
            key = (text, tone_status, tone)
            job = jobs_by_key.get(key)
            if job is None:
                job = AudioJob(
                    key=key,
                    text=text,
                    provider=provider,
                    voice=voice,
                    language_code=language_code,
                    filename=audio_filename(provider, voice, language_code, text, tone_status, tone),
                    pronunciation_source=pron.get("source") if pron else None,
                    tone_status=tone_status,
                    tone=tone,
                )
                jobs_by_key[key] = job
            job.article_ids.append(article_id)
            source_lemma_id = lemma.get("source_lemma_id")
            if isinstance(source_lemma_id, int):
                job.source_lemma_ids.append(source_lemma_id)

    return [jobs_by_key[key] for key in sorted(jobs_by_key)]


def _is_expression(lemma: dict[str, Any], text: str) -> bool:
    return bool(lemma.get("is_sub_article")) or " " in text


def _best_pronunciation_for_lemma(lemma: dict[str, Any], text: str) -> dict[str, Any]:
    for form in lemma.get("word_forms", []):
        if not isinstance(form, dict) or form.get("word_form") != text:
            continue
        pron = form.get("pronunciation") or []
        if pron and isinstance(pron[0], dict):
            return pron[0]
    return {}
```

```python
def write_audio_enriched_lemmas(lemma_dir: Path, output_dir: Path, jobs: list[AudioJob]) -> int:
    jobs_by_key = {job.key: job for job in jobs}
    changed_files = 0

    for path in sorted(lemma_dir.glob("*.json")):
        data = json.loads(path.read_text(encoding="utf-8"))
        changed = False
        for lemma in data.get("lemmas", []):
            if not isinstance(lemma, dict):
                continue
            text = normalize_audio_text(str(lemma.get("lemma") or ""))
            pron = _best_pronunciation_for_lemma(lemma, text)
            job = jobs_by_key.get((text, pron.get("tone_status") if pron else None, pron.get("tone") if pron else None))
            if job is None:
                continue
            item = audio_metadata(job)
            if lemma.get("audio") != {"lemma": [item]}:
                lemma["audio"] = {"lemma": [item]}
                changed = True
        if changed:
            output_dir.mkdir(parents=True, exist_ok=True)
            (output_dir / path.name).write_text(json.dumps(data, ensure_ascii=False, indent=2) + "\n", encoding="utf-8")
            changed_files += 1

    return changed_files


def audio_metadata(job: AudioJob) -> dict[str, Any]:
    item: dict[str, Any] = {
        "type": "tts",
        "provider": job.provider,
        "voice": job.voice,
        "language_code": job.language_code,
        "format": "mp3",
        "file": job.filename,
        "text": job.text,
        "source": "generated",
        "content_sha256": "",
    }
    if job.pronunciation_source is not None:
        item["pronunciation_source"] = job.pronunciation_source
    if job.tone_status is not None:
        item["tone_status"] = job.tone_status
    if job.tone is not None:
        item["tone"] = job.tone
    return item
```
